Problem Description

We are trying to create a computer representation of a fluid that moves and behaves like a fluid in real life. This is important because it helps test how aerodynamic planes and cars are, lets us predict the weather, and build infrastructure like dams and fluid piping. Without computer simulations, all models would have to be physically prototyped, which is difficult and expensive. Fluid dynamics is challenging because fluids are chaotic, meaning that small changes in initial conditions result in very different results. And there are a lot of parameters that all affect each other. Not only that, we don’t have the mathematical methods to directly compute solutions to the equations that describe how fluids behave, so we have to use simulation. Although fluids in real life are not incompressible, many applications of fluids do not rely on the compressibility of fluids and simulating an incompressible fluid is often a good approximation of the reality. Furthermore, this tradeoff is often taken when needing real time performance.


Goals and Deliverables

Questions to answer: How do we efficiently find the nearest neighbors for the incompressibility constraints? What is a good-looking yet fast method of rendering a liquid from particles?

Plan to deliver: Real-time interactive simulation of a 2d fluid with at least 30 fps. This will be a basic simulation of water falling into a box, and we will have a demo to add and swirl fluid in real time. We plan to simulate this by simulating many particles, and creating a shader that will color the area covered by these particles with water. We could also add rigid objects to show splashing (ripples) and buoyancy. We can measure quality/performance by using fps, resolution, and perhaps number of particles as metrics. These kinds of fluids are commonly used in video games, which is why we think it is feasible to make such a demonstration.

Hope to deliver: If things go well and we get ahead of schedule, there are a couple of directions we can go in. One is fluid simulation in a beer bottle, with the bottle in various lighting environments. The fluid is also carbonated, so the liquid would have bubbles rising up. Another direction we could go is simulating a ferrofluid, which could be done by assigning a magnetic field to each particle, and creating magnetic rigid objects.
